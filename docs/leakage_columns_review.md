# 누수 컬럼 검토 결과 (Week 1)

> 작성: 심재형 (PM) | 검증 협력: 이고은 (Dev A) | 기준일: 2026-05-03

## 확정 DROP (이고은 검증)

| 컬럼 | 이유 |
|------|------|
| `reservation_status` | `is_canceled`와 정의상 동치. 크로스탭 위반 0건으로 100% 일치 확인. |
| `reservation_status_date` | 도착일 이후에도 갱신되는 시점값. 정상 체크인건은 체크아웃 시점, 취소건은 취소 처리 시점으로 기록 — 어느 쪽이든 미래 정보. |

## PM 임시 결정 DROP

| 컬럼 | 이유 |
|------|------|
| `assigned_room_type` | 체크인 시점에 배정되는 값. 취소건은 체크인하지 않으므로 reserved와 동일한 기본값으로 채워진 가능성 높음. `is_canceled=0` 그룹의 불일치율이 압도적으로 높을 경우 누수 확정. |
| `previous_bookings_not_canceled` | `is_repeated_guest=1`인 행 중 이 값이 0인 케이스 25.5% 존재 — 재방문자인데 성공 이력 없다는 모순. 신뢰도 낮아 모델에 노이즈 줄 위험. |
| `days_in_waiting_list` | 타인 취소로 대기가 해제된 경우, 대기일 자체에 미래 취소 정보가 간접 반영됨. 발표 어려움 ① 추가 사례로 활용. |

## 조건부 KEEP (Week 2 EDA 재검증 예정)

| 컬럼 | 현황 | 재검증 항목 |
|------|------|------------|
| `previous_cancellations` | 연도별 비율 감소 추세 비직관적(2015: 20% → 2017: 0.7%). 논리적 모순은 없음. 취소율 신호 강력(≥1이면 91%). | Week 2 EDA에서 연도별 분포 + 취소율 상관 재확인 |

## KEEP (그대로 사용)

| 컬럼 | 처리 |
|------|------|
| `booking_changes` | 예약 시점 이전 변경 이력. 중요 예측 신호. |
| `agent` | NaN → 0 (여행사 미경유), 값 있음 → 1 (인디케이터 변환) |
| `company` | NaN → 0 (법인 예약 아님), 값 있음 → 1 (인디케이터 변환) |
