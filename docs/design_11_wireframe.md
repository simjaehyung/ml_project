## 1. 탭 1 — 기존 예약 취소 위험 우선순위 리스트

> **사용 시점:** 매니저가 매일 아침 / 점심에 "오늘·내일 도착 예약 중 위험 높은 것 누구?" 묻는 시점
> **모델 입력:** 테스트셋(미래 예약 풀) 전체
> **모델 출력:** 위험도 정렬 표 + 행별 SHAP 근거 + 권장 액션

### 1-1. 화면 레이아웃

```
┌───────────── 필터 영역 ──────────────────────────────┐
│ 호텔 [전체 ▾]  도착일 [2017-03-15 ~ 2017-03-22]      │
│ 위험도 임계값 [────●──── 0.65]   [필터 적용]         │
└───────────────────────────────────────────────────────┘

┌───────────── 요약 메트릭 (st.columns 3) ─────────────┐
│  대상 예약    고위험 예약    예상 취소 수             │
│   1,247       342 (27%)      ~289                     │
└───────────────────────────────────────────────────────┘

┌───────────── 위험 순위 표 (st.dataframe) ────────────┐
│ # │ 예약 ID │ 호텔 │ 도착 │ LT │ ADR  │ 위험 │ 액션  │
│ 1 │ R-78321 │ City │ 3-15 │220 │ 95€  │ 96%  │ 콜+오 │
│ 2 │ R-78415 │ City │ 3-15 │189 │ 110€ │ 92%  │ 콜+오 │
│ 3 │ R-78127 │ Resort│3-16 │ 75 │ 145€ │ 88%  │ 콜    │
│ ... 페이지네이션                                       │
└───────────────────────────────────────────────────────┘

(행 클릭 → 아래에 상세 패널 펼침)

┌───────────── 상세 패널 (st.expander) ────────────────┐
│ 예약 R-78321 — 위험도 96% (HIGH)                     │
│                                                       │
│ 주도 변수 Top 3 (SHAP):                              │
│   • lead_time = 220일이 매우 김 (+0.21)              │
│   • previous_cancellations = 1회 이력 (+0.15)        │
│   • deposit_type = Non Refund (+0.12)                │
│                                                       │
│ 권장 액션 (action_recommender.py):                   │
│   • 도착 30일 전 재확인 콜                            │
│   • 선결제 전환 유도 안내 발송                        │
│   • 오버부킹 후보로 분류 (탭 2 Flexi 풀에 자동 등록) │
│                                                       │
│ [상세 닫기]                                           │
└───────────────────────────────────────────────────────┘
```

### 1-2. 사용 위젯 매핑

| 영역 | Streamlit 함수 | 비고 |
|---|---|---|
| 호텔·날짜·임계값 필터 | `st.selectbox`, `st.date_input`, `st.slider` | 사이드바 또는 상단 |
| 요약 메트릭 3개 | `st.columns(3)` + `st.metric` | STEP 1 패턴 ④ |
| 위험 순위 표 | `st.dataframe` | STEP 1 패턴 ② |
| 상세 패널 펼침 | `st.expander` | 행 선택 후 표시 |
| 위험도 색상 강조 | `st.dataframe` 내 `.style.background_gradient` | 시각적 우선순위 |

### 1-3. 데이터 의존성

| 컬럼 | 출처 | 누구 |
|---|---|---|
| 예약 ID | test_processed.csv (인덱스) | 이고은 |
| 호텔, 도착일, lead_time, ADR | test_processed.csv | 이고은 |
| 취소 위험 % | XGBoost `predict_proba` | 김나리 |
| 주도 변수 Top 3 | SHAP values | 김나리 |
| 권장 액션 | `src/action_recommender.py` | 심재형 |

## 2. 탭 2 — 신규 예약 Flexi 라우팅 권장

> **사용 시점:** 신규 예약 들어왔을 때 매니저가 "이건 Standard에 둘까 Flexi 풀에 넣을까?" 결정하는 시점
> **모델 입력:** 매니저가 입력 폼에 채운 신규 예약 1건
> **모델 출력:** 위험도 + SHAP + Standard/Flexi 권장 + 권장 할인율

### 2-1. 화면 레이아웃

```
┌───────────── 신규 예약 입력 폼 (st.form) ────────────┐
│  호텔        [City Hotel ▾]                          │
│  도착일      [2017-03-22]                            │
│  체크아웃   [2017-03-25]                             │
│  성인 / 아동 [2] [0]                                 │
│  Lead Time   [180]   ADR (€) [120.00]                │
│  객실 종류   [A ▾]                                    │
│  Market Seg. [Online TA ▾]                           │
│  과거 취소   [0]    재방문   [☐]                     │
│                                                       │
│        [위험도 예측 + 라우팅 권장]                   │
└───────────────────────────────────────────────────────┘

(제출 시 결과 영역 펼침)

┌───────────── 결과 — 메트릭 카드 (st.columns 3) ──────┐
│  취소 위험         권장 라우팅       권장 할인율      │
│   78%              Flexi              12.3%           │
│   ↑ HIGH           [Standard로 변경]                  │
└───────────────────────────────────────────────────────┘

┌───────────── 이야기 (story_generator.py) ─────────────┐
│ "이 예약은 lead_time이 180일로 매우 길고, 과거 취소  │
│  이력이 1회 있으며, 보증금 정책이 Non Refund 입니다.│
│  과거 유사 패턴에서 96% 가 취소되었습니다."          │
└───────────────────────────────────────────────────────┘

┌───────────── 주도 변수 (SHAP waterfall) ──────────────┐
│ baseline: 0.37                                        │
│   + lead_time 180일      → +0.21                      │
│   + Non Refund deposit   → +0.12                      │
│   + previous_cancel ≥1   → +0.15                      │
│   - market_segment Direct→ -0.04                      │
│ ─────────────────                                      │
│ 최종 위험도: 0.78                                     │
└───────────────────────────────────────────────────────┘

┌───────────── 매니저 결정 (GDPR Art.22) ──────────────┐
│ 자동 라우팅 권장: Flexi                              │
│ 매니저 최종 결정 → [✓ Flexi로 확정] [Standard 유지] │
│                                                       │
│ ℹ 자동 결정 단독 발효 금지. 매니저 승인 필수.        │
└───────────────────────────────────────────────────────┘
```

### 2-2. 사용 위젯 매핑

| 영역 | Streamlit 함수 | 비고 |
|---|---|---|
| 입력 폼 | `st.form` + `st.number_input`, `st.selectbox`, `st.date_input` | STEP 1 패턴 ③ |
| 결과 메트릭 카드 | `st.columns(3)` + `st.metric` | STEP 1 패턴 ④ |
| 이야기 | `st.markdown` 또는 `st.info` | story_generator 출력 |
| SHAP waterfall | `streamlit-shap` 또는 `st.pyplot(shap_plot)` | Dev B SHAP 출력과 연결 |
| 매니저 결정 버튼 | `st.button` × 2 | GDPR Art.22 명시 |
| 주의 안내 | `st.caption` 또는 `st.warning` | 자동 결정 단독 금지 명시 |

### 2-3. 데이터 의존성

| 항목 | 출처 | 누구 |
|---|---|---|
| 위험도 % | 폼 입력 → XGBoost `predict_proba` | 김나리 (모델) + 이고은 (UI 연결) |
| SHAP 값 | 폼 입력 → SHAP explainer | 김나리 |
| 이야기 | SHAP Top 3 → `src/story_generator.py` | 심재형 |
| 라우팅 권장 | 위험도 + 임계값 (design_06 기반) | 심재형 |
| 권장 할인율 | `5% + (위험점수 - 0.5) × 26%` (5~18% 클립) | 이고은 (이미 STEP 1에 구현) |
