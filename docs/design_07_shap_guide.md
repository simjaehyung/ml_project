## 1. 출발점 — 게임 이론의 Shapley 값

**배경**: 1953년 Lloyd Shapley 제안, 2012년 노벨 경제학상

**문제 설정**: 여러 명이 협력해서 어떤 결과를 냈을 때, 각자의 기여도에 따라 결과를 공정하게 배분하는 방법은?

**핵심 아이디어**: 한 플레이어의 기여도 = 그 플레이어가 **없는** 팀과 **있는** 팀의 결과 차이. 단, 팀 구성의 **모든 가능한 순서**에 걸쳐 평균을 낸다.

```
φᵢ = Σ  [|S|!(|N|-|S|-1)! / |N|!] × [v(S ∪ {i}) - v(S)]
    S⊆N\{i}
```

- `N` = 전체 플레이어 집합
- `S` = i를 제외한 부분집합
- `v(S ∪ {i}) - v(S)` = i가 팀 S에 합류했을 때의 결과 변화량
- 앞의 계수 = 그 순서로 팀이 구성될 확률 (가중치)

## 2. ML에 적용 — SHAP (Lundberg & Lee, 2017)

| 게임 이론 | ML 적용 |
|----------|--------|
| 플레이어 | 피처 |
| 게임 결과 | 모델 예측값 |
| 배분할 총량 | `f(x) - E[f(X)]` (예측값 - 전체 평균 예측값) |

"평균에서 얼마나 벗어났는가"를 각 피처의 기여분으로 분해한다.

### 핵심 보장 (Efficiency)

모든 피처의 SHAP 값을 더하면 반드시 `f(x) - E[f(X)]`가 된다.

```
φ₁ + φ₂ + ... + φₙ = f(x) - E[f(X)]
```

**예시**: 취소 확률 0.78, 전체 데이터 평균 0.37
→ SHAP 값 합산 = +0.41

```
lead_time          +0.21  ──┐
prev_cancellation  +0.15    │
deposit_type       +0.09    ├── 합산 = +0.41
weather_bad        +0.01    │
is_group           -0.05  ──┘
```

## 3. "피처가 없다"는 것을 어떻게 표현하는가

Shapley 계산에서 피처 일부만 "있는" 상황을 만들어야 하지만, 실제 모델은 항상 전체 피처를 요구한다.

**해결**: 없는 피처 = 배경 데이터셋(background data)에서 샘플링해서 채운다.

> "이 피처를 모른다" ≡ "이 피처를 학습 데이터 분포에서 무작위로 가져온다"

이 처리 방식이 SHAP 구현체마다 달라지고, 배경 데이터셋 선택에 따라 SHAP 값이 조금씩 달라질 수 있다.

---

## 4. 주요 구현체

| 구현체 | 대상 모델 | 계산 방식 | 속도 |
|--------|----------|----------|------|
| **TreeSHAP** | XGBoost, LightGBM, RandomForest | 정확한 계산 (다항시간) | 빠름 |
| **KernelSHAP** | 모든 모델 | 샘플링 기반 근사 | 느림 |
| **LinearSHAP** | 선형 모델 | 정확한 계산 | 매우 빠름 |
| **DeepSHAP** | 신경망 | DeepLIFT 기반 근사 | 중간 |

트리 기반 모델(XGBoost / LightGBM / RandomForest)을 사용할 경우 **TreeSHAP**이 적용된다. 샘플링 없이 정확하게 계산되며, 지수 시간이 아닌 다항시간 `O(TLD²)` (T=트리 수, L=리프 수, D=깊이)으로 동작한다.

## 5. SHAP이 유일하게 만족하는 4가지 공리

이 네 가지를 동시에 만족하는 Attribution 방법은 Shapley 값뿐임이 수학적으로 증명되어 있다. LIME, 단순 피처 중요도 등 다른 방법은 이 중 하나 이상을 만족하지 못한다.

| 공리 | 의미 |
|------|------|
| **Efficiency** | 모든 SHAP 값의 합 = 예측값 - 기준값 (설명이 완전하게 분해됨) |
| **Symmetry** | 두 피처가 모든 coalition에서 동일한 기여를 하면 동일한 SHAP 값을 받는다 |
| **Dummy** | 어떤 coalition에서도 예측을 바꾸지 않는 피처의 SHAP = 0 |
| **Additivity** | 두 모델의 합산 예측에서 SHAP = 각 모델 SHAP의 합 |

## 6. 로컬 vs 글로벌 해석

### 로컬 SHAP (per-prediction)

예약 **한 건**에 대한 설명. 저희 시스템의 핵심 사용처.

> "이 예약의 취소 확률이 0.78인 이유는 lead_time이 길어서(+0.21), 이전 취소 이력이 있어서(+0.15), ..."

### 글로벌 SHAP (전체 데이터 집계)

전체 데이터에서 `|SHAP값|`의 평균 → 피처 중요도로 활용.

```
피처 중요도 = (1/n) Σ |φᵢ(xⱼ)|  for j=1..n
```

단순 피처 중요도(불순도 기반 등)보다 신뢰도가 높다. 특히 트리 모델의 기본 feature importance는 데이터 분포에 편향되기 쉬운 반면, SHAP 기반 중요도는 실제 예측 기여를 반영한다.

## 7. 실전에서 알아야 할 주의점

### 상관된 피처 문제

두 피처가 강하게 상관되어 있으면 SHAP이 기여도를 둘 사이에 나눠버린다.

예: `lead_time`과 `booking_changes`가 상관되어 있으면 둘 다 SHAP 값이 낮게 나올 수 있다. 이를 "중요하지 않다"로 오해하면 안 된다. 상관 피처 그룹을 묶어서 해석하는 것이 더 정확하다.

### SHAP ≠ 인과관계

SHAP은 "이 피처가 이 예측에 얼마나 기여했는가"를 말할 뿐, "이 피처가 취소를 유발했는가"는 말하지 않는다. 모델이 학습한 상관 패턴 기반이다.

### 배경 데이터셋 의존성

`E[f(X)]` 계산과 "없는 피처" 처리가 배경 데이터셋에 따라 달라진다. 전체 학습 데이터를 배경으로 쓸지 샘플을 쓸지에 따라 SHAP 값이 조금씩 달라질 수 있으므로, 실험 간 비교 시 배경 데이터셋을 통일해야 한다.

### TreeSHAP의 개입적(interventional) 특성

TreeSHAP은 기본적으로 interventional perturbation을 사용한다. 즉, 피처를 "없앨 때" 조건부 분포가 아닌 주변 분포(marginal distribution)에서 샘플링한다. 상관 피처가 많은 경우 이 특성이 attribution에 영향을 줄 수 있다.

## 8. 저희 시스템에서의 활용 흐름

```
예약 1건 입력
      ↓
  모델 예측 (f(x) = 0.78)
      ↓
  TreeSHAP 계산
      ↓
  기여 요인 분해
  ┌─────────────────────────────┐
  │ lead_time        +0.21  ▲  │
  │ prev_cancel      +0.15  ▲  │
  │ deposit_type     +0.09  ▲  │
  │ weather          +0.01  ▲  │
  │ is_group         -0.05  ▼  │
  │ ─────────────────────────  │
  │ 합산              +0.41    │
  │ 기준값            +0.37    │
  │ 예측값             0.78    │
  └─────────────────────────────┘
      ↓
  매니저에게 표시 + 권장 행동 연결
```
